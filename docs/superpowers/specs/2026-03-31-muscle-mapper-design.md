# 근육 좌표 매핑 툴 설계

## 배경

`muscles.ts`의 `area` 필드(top/left/width/height %)는 이미지 위 클릭 영역을 정의한다.
현재 상체(16개)만 수작업으로 입력된 상태이고, 하체/척추/복부(22개+)는 미매핑.
매핑 작업이 번거로운 이유: 이미지를 눈으로 보면서 퍼센트를 추정 → 직접 입력 → 확인 → 수정 반복.

**목표**: `/admin/mapper` 페이지에서 이미지를 클릭/드래그하면 좌표가 자동으로 계산되어 복사 가능한 형태로 출력. 근육 1개당 5초 이내로 매핑 완료.

---

## 기능 정의

### 1. 카테고리 탭
- 상체 / 하체 / 척추 / 복부 및 호흡 탭
- 탭 전환 시 해당 카테고리 이미지(CATEGORY_IMAGES)와 기존 매핑 데이터 로드

### 2. 이미지 인터랙션
두 가지 모드:

**클릭 모드** (단순 중심점)
- 이미지 위 클릭 → 해당 위치의 `top`, `left` 퍼센트 기록
- 기본 width: `12%`, height: `3%` (현재 상체 평균값)
- 빠르게 중심점만 찍고 싶을 때

**드래그 모드** (영역 직접 지정)
- mousedown → mousemove → mouseup으로 사각형 그리기
- `top`, `left`, `width`, `height` 모두 자동 계산
- 드래그 중 반투명 파란 사각형 프리뷰 표시

### 3. 출력 패널
```
{ top: '33.5%', left: '84.5%', width: '13%', height: '3%' }
```
- 클릭/드래그 완료 즉시 갱신
- [복사] 버튼 → 클립보드 복사 + "복사됨 ✓" 피드백

### 4. 기존 매핑 오버레이
- 해당 카테고리의 이미 `area`가 있는 근육들을 이미지 위에 표시
- 색상: 연한 파란 반투명 (`bg-sky-400/30 border border-sky-400`)
- 호버 시 근육 이름 툴팁
- "어디까지 됐는지" 시각적으로 확인 가능

## 아키텍처

### 파일
- `src/app/admin/mapper/page.tsx` — 단일 파일, `'use client'`

### 데이터 흐름
```
CATEGORY_IMAGES (기존) → 이미지 src
allMuscles (기존) → 기존 매핑 오버레이 필터링
mouseEvent → (x/containerWidth * 100).toFixed(1)+'%' → 출력
```

### 좌표 계산
```ts
const rect = containerRef.current.getBoundingClientRect();
const x = ((e.clientX - rect.left) / rect.width * 100).toFixed(1) + '%';
const y = ((e.clientY - rect.top) / rect.height * 100).toFixed(1) + '%';
```
